# 워크위드 온보딩 체계

> 작성일: 2026-03-09
> 목적: 누가 와도 "이거 읽고 이대로 해"가 가능한 체계
> 대상: 모든 신규 입사자 + 기존 직원 재교육

---

## 1. 온보딩 전체 구조

```
[공통 온보딩] — 누가 오든 동일 (1~2일)
  ↓
[역할별 온보딩] — 직무에 따라 분기 (3일~2주)
  ├── A. 현장관리자 (과장)
  ├── B. 영업·사무 (매니저/팀장)
  └── C. CS팀 (콜센터)
  ↓
[실전 투입] — OJT + 매일 체크
  ↓
[평가] — 2주/1개월/3개월 시점
```

## 3. 역할별 온보딩

### 3-A. 현장관리자 (과장) 온보딩 — 2주

**교육 자료:**
- 현장관리부 업무 지침서 (docs/site-management-guide.html) — 10개 챕터
- 현장관리 업무 양식 (docs/site-management-templates.html)

**일정:**

| 일차 | 내용 | 방법 | 담당 |
|------|------|------|------|
| Day 3 | 매뉴얼 인쇄본 전달 + 1~3장 읽기 | 자습 → 질의응답 | 이사님 |
| Day 4 | 매뉴얼 4~6장 읽기 + 거래처 현황 설명 | 자습 → 질의응답 | 이사님 |
| Day 5 | 매뉴얼 7~10장 + 양식 사용법 | 자습 → 질의응답 | 이사님 |
| Day 6~7 | 이사님과 현장 동행 (인원 투입 참관) | OJT | 이사님 |
| Day 8~9 | 이사님 감독 하에 인원 투입 직접 실행 | OJT | 이사님 |
| Day 10 | 단독 인원 투입 시도 (이사님 대기) | 실전 | 이사님 확인 |
| 2주차 | 거래처 현장 순회 동행 → 단독 순회 | OJT → 실전 | 이사님 |

**핵심 교육 포인트:**

```
1. 매일 인원 투입/배치 (최우선)
   - 시트에서 내일 투입 인원 확인
   - 각 거래처 담당자에게 연락
   - 결원 발생 시 즉시 대체 인력 배치

2. TO 관리
   - TO 발생 → 24시간 내 구인 시작
   - 3일 미충원 → 이사님/대표 보고
   - 절대 방치 금지

3. 현장 순회
   - 주 3회+ 현장 방문
   - 반장 면담, 근로자 상태 확인
   - 거래처 담당자 인사

4. 보고 체계
   - 매일 카톡: 현장 현황 + 이슈
   - 이슈 발생 → 즉시 전화
   - 주간: 거래처별 현황 정리

5. 기본 원칙
   - "할 일 없으면 현장 가라"
   - "모르면 물어봐라 (혼자 판단 X)"
   - "보고 안 하는 게 가장 큰 문제"
```

**평가 시점:**

| 시점 | 기준 | 통과/실패 |
|------|------|----------|
| 2주차 | 인원 투입 단독 가능 | 가능 → 이사님 손 떼기 |
| 1개월 | 알아서 하는 사람인가 | 시켜야 하면 → 관리 강화 |
| 3개월 | 현장 안정 + TO 즉시 대응 | 실패 → 교체 검토 |

### 3-B. 영업·사무 (매니저/팀장) 온보딩 — 1주

**교육 자료:**
- 영업응대 온보딩 가이드 (docs/walkwith-onboarding.html) — 인터랙티브
- 영업문의 응대 매뉴얼 (MD) — 상세 매뉴얼
- 영업문의 응대 분기 트리 (MD) — 판단 플로우
- 견적 시뮬레이터 (docs/quotation-tool.html)

**일정:**

| 일차 | 내용 | 방법 | 담당 |
|------|------|------|------|
| Day 3 | 온보딩 가이드 HTML 학습 (서비스 분류) | 자습 + 이사님 설명 | 이사님 |
| Day 4 | 전화 응대 실습 (체크리스트 10항목) | 롤플레이 | 이사님 |
| Day 5 | 견적 시뮬레이터 사용법 + 시트 기록 방법 | 실습 | 이사님 |
| Day 6 | 실전 전화 1건 (이사님 옆에서) | OJT | 이사님 |
| Day 7 | 단독 접수 시작 + 일일 피드백 | 실전 | 이사님 확인 |

**핵심 교육 포인트:**

```
1. 전화 접수 (체크리스트)
   □ 업체명
   □ 담당자 성함/연락처
   □ 필요 인원 수
   □ 업종/직종
   □ 근무 지역
   □ 근무 시간/기간
   □ 희망 시작일
   □ 급여 수준 (최저임금 기준)
   □ 숙소/식사 제공 여부
   □ 기타 요청사항

2. 시트 기록
   - 접수 즉시 구글시트에 기록
   - "다음 액션" 칸 반드시 기입
   - 빈칸 절대 금지

3. 서비스 분류 (직접 판단 X → 이사님 판단)
   - 접수 → 체크리스트 채움 → 시트 기록 → 이사님 확인
   - "안 됩니다" 절대 금지
   - 모르면 "확인 후 연락드리겠습니다"

4. 팔로업 (파견/채용대행)
   - 견적 발송 +3일: 전화 1통
   - +7일: 미응답 시 문자
   - +14일: 최종 → 보류/종결

5. 견적 발송 (파견/채용대행)
   - 6%+ 기계적 발송 (판단 불필요)
   - 도급 견적은 이사님 판단 후 발송
```

**평가 시점:**

| 시점 | 기준 | 통과/실패 |
|------|------|----------|
| 1주 | 체크리스트 누락 없이 기록 | 누락 반복 → 재교육 |
| 2주 | "안 됩니다" 0회 + 시트 100% 기입 | 미달 → 분기 B (사무 전환) |
| 1개월 | 팔로업 이행률 80% | 미달 → 역할 재조정 |

### 3-C. CS팀 (콜센터) 온보딩 — 1주

**교육 자료:**
- 콜센터 운영 매뉴얼 (TODO — 미완성)
- 콜쉐어링 단가표
- 상담 스크립트 (거래처별)

**일정:**

| 일차 | 내용 | 방법 | 담당 |
|------|------|------|------|
| Day 3 | 콜센터 사업 구조 (전담/쉐어링/CS대행) | 이사님 설명 | 이사님 |
| Day 4 | 단가 체계 + 견적 방법 | 실습 | 이사님/이고은 |
| Day 5 | 상담 프로세스 + 품질 기준 | 이고은 설명 | 이고은 |
| Day 6 | 기존 거래처 상담 참관 | OJT | 이고은 |
| Day 7 | 영업 계획 수립 (신규 거래처 개발) | 자율 | 이사님 확인 |

**핵심 교육 포인트:**

```
1. 콜센터 서비스 3가지
   - 전담: 1개 업체 전용 상담원 배치
   - 쉐어링: 여러 업체 공유 (시간/콜 분배)
   - CS대행: 이커머스 CS 아웃소싱

2. 단가 체계 (확정)
   ~20콜/일: 50만원/월 (쉐어링)
   ~50콜/일: 80만원/월 (쉐어링)
   ~100콜/일: 150만원/월 (반전담)
   ~200콜/일: 300만원/월 (전담)

3. 팀장 역할 = 영업
   - 신규 거래처 개발이 핵심
   - 이고은 센터장은 운영 집중
   - 3개월 내 BEP (월 300만원+) 목표

4. 보고 체계
   - 주간: 영업 활동 보고 (컨택수, 견적수)
   - 월간: 매출/비용 보고
```

## 5. 온보딩 체크리스트 (관리자용)

### 5-1. 입사 전 준비

```
□ 근로계약서 준비
□ 4대보험 취득 신고 (홍슬기)
□ 구글시트 접근 권한 부여
□ 카톡 그룹 초대
□ 사무실 자리/장비 배정
□ 온보딩 자료 인쇄 (역할별)
□ 교육 담당자 지정 (이사님 or 대표)
```

### 5-2. 공통 온보딩 완료 확인

```
□ 회사 소개 받음
□ 서비스 분류 이해 (퀴즈 통과)
□ 조직 구조 + 보고 체계 이해
□ 시스템 사용 가능 (시트, 전화, 카톡)
□ 취업규칙 수령 확인
```

### 5-3. 역할별 온보딩 완료 확인

**현장관리자:**
```
□ 매뉴얼 10개 챕터 읽기 완료
□ 인원 투입 프로세스 이해
□ 거래처별 담당자·특이사항 파악
□ 현장 동행 2회+ 완료
□ 인원 투입 단독 1회+ 완료
□ 양식 사용법 숙지
```

**영업·사무:**
```
□ 온보딩 가이드 HTML 학습 완료
□ 체크리스트 10항목 암기
□ 전화 응대 롤플레이 통과
□ 견적 시뮬레이터 사용 가능
□ 시트 기록 실습 완료
□ 실전 전화 1건+ 완료
```

**CS팀:**
```
□ 콜센터 서비스 3가지 이해
□ 단가 체계 숙지
□ 상담 참관 1일+ 완료
□ 영업 계획 초안 작성
```

## 6. 기존 직원 재교육 (30분)

```
[대상] 최한솔, 홍슬기 (기존 직원)

[방법]
  1. 온보딩 가이드 HTML 같이 열어놓고 설명 (15분)
  2. 매뉴얼 핵심 부분 읽기 (10분)
  3. "전화 옆에 붙여두기" 인쇄 배포 (5분)

[1주일 후 확인]
  □ 체크리스트 누락 건 체크
  □ "안 됩니다" 발언 여부
  □ 팔로업 기록 확인
  □ 시트 "다음 액션" 기입률
```

## 7. 온보딩 실패 시 대응

| 역할 | 실패 징후 | 시점 | 대응 |
|------|----------|------|------|
| 현장관리자 | 투입 단독 불가, TO 방치 | 2주~1개월 | 관리 강화 → 3개월 재평가 → 교체 |
| 영업·사무 | 체크리스트 누락, "안 됩니다" 반복 | 2주 | 사무 전환 (분기 B) → 1개월 재평가 |
| CS팀장 | 영업 활동 0, 매출 무성장 | 3개월 | BEP 미달 시 재면담 → 교체 검토 |

```
공통 실패 패턴:
  - "안 배웠어요" → 자료 있는데 안 읽은 것 → 태도 문제
  - "몰랐어요" → 보고 안 한 것 → 보고 체계 재교육
  - "시간이 없어요" → 업무 우선순위 문제 → 역할 재조정
```
