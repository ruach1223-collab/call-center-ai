GPS 출퇴근 시스템 설정 가이드
워크위드 — 설정 완료까지 약 15분

1 구글 스프레드시트 만들기

sheets.new 접속 → 새 스프레드시트 생성 → 이름: "GPS출퇴근"

탭 1: "근로자" (시트 하단 탭 이름 변경)

A: 토큰B: 이름C: 전화번호D: 소속현장E: 기기IDF: 링크
test001테스트010-0000-0000서울식품(빈칸 — 자동입력됨)(수식)

F2 수식 (링크 자동생성):

="https://ruach1223-collab.github.io/call-center-ai/gps-checkin.html?t="&A2

F2에 입력 후 아래로 드래그하면 전체 적용됩니다.

토큰 규칙: 영문+숫자 조합 추천 (예: kim01, park02, nguyen03)
한 사람당 하나의 고유 토큰. 카톡으로 F열의 링크를 보내면 됩니다.

탭 2: "현장" (+ 버튼으로 새 탭 추가)

A: 현장명B: 위도C: 경도D: 반경(m)
서울식품37.3219126.8311300
유니투스37.2851127.0103200
위도/경도 찾기: 구글맵에서 현장 주소 검색 → 지도 우클릭 → 좌표 복사
반경: 현장 크기에 따라 200~500m 설정. 이 안에 있으면 "정상", 밖이면 "현장외"

탭 3: "체크인기록" (자동으로 채워짐)

A: 날짜B: 이름C: 구분D: 시간E: 위도F: 경도G: 현장명H: 거리(m)I: 상태
1행에 헤더만 입력. 데이터는 출퇴근 시 자동 추가됨
중요: 1행에 반드시 헤더(날짜, 이름, 구분, 시간, 위도, 경도, 현장명, 거리, 상태)를 입력하세요. 빈 시트면 코드가 오류납니다.

2 Apps Script 배포

방금 만든 구글시트에서: 확장프로그램 → Apps Script 클릭

  1. 기존 코드 전체 삭제
  2. docs/gps-checkin-apps-script.js 파일 내용 전체 복사 → 붙여넣기
  3. 저장 (Ctrl+S)
  4. 상단 메뉴: 배포 → 새 배포
  5. 유형: 웹 앱 선택
  6. 실행 주체: 나
  7. 액세스 권한: 모든 사용자
  8. 배포 클릭 → URL 복사
배포 URL 형태: https://script.google.com/macros/s/AKfycb.../exec
이 URL을 다음 단계에서 사용합니다.

3 체크인 페이지에 URL 연결

GitHub 저장소에서 gps-checkin.html 파일 열기:

// 이 줄을 찾아서:
const SCRIPT_URL = 'YOUR_APPS_SCRIPT_URL_HERE';

// 배포 URL로 교체:
const SCRIPT_URL = 'https://script.google.com/macros/s/AKfycb.../exec';

수정 후 커밋 → 푸시하면 GitHub Pages에 반영됩니다.

4 테스트

  1. 근로자 탭에 테스트 데이터 입력 (토큰: test001, 이름: 테스트)
  2. 폰에서 F열 링크 접속
  3. 위치정보 동의 → 출근 버튼 클릭
  4. 구글시트 "체크인기록" 탭에 데이터가 추가되면 성공!
테스트 체크리스트:
  • 출근 클릭 → 체크인기록에 기록 확인
  • 출근 다시 클릭 → "이미 출근 처리됨" 메시지 확인
  • 퇴근 클릭 → 퇴근 기록 추가 확인
  • 다른 기기(PC)에서 같은 링크 → 기기 불일치 상태 확인
  • 페이지 하단 → 이번 달 출근 이력 표시 확인

5 RAG 시스템 연동 (대시보드 + SMS)

5-1. 구글시트 ID 등록

스프레드시트 URL에서 ID 복사:

https://docs.google.com/spreadsheets/d/여기가_시트_ID/edit

rag-system/app/sheets.js에서 GPS_CHECKIN_SPREADSHEET_ID_HERE를 교체

5-2. 알리고 SMS 환경변수 (.env)

ALIGO_API_KEY=발급받은_API키
ALIGO_USER_ID=알리고_아이디
ALIGO_SENDER=발신번호(사전등록필수)

5-3. Docker 재빌드

cd rag-system && docker-compose down && docker-compose up -d --build

! 당일 결근 시 대체인력 구인 로직

대체인력 후보 선정 기준 (자동)

구분조건표시 정보
최근 근무자 최근 1개월 내 근무 이력 있음
+ 오늘 출근 기록 없음
+ 지역 필터 적용 가능		 이름, 최근 근무지, 사는 곳, 근무일수, 마지막 근무일, 연락처
최근 지원자 채용일이 있는 전체명단
+ 지역 필터 적용 가능		 이름, 사는 곳, 채용일, 연락처

문자 발송 흐름

  1. 대시보드에서 결근 현장 확인 → "대체인력 연락하기" 클릭
  2. 대체 가능 인력 목록에서 10명 이상 체크 (2~3명만 보내면 전부 거절 위험)
  3. 현장명 입력 (자동 입력됨)
  4. "문자 보내기" 클릭
  5. 확인 팝업: 이름, 생년월일, 연락처, 사는곳 목록 확인
  6. "문자 발송 확인" → 알리고 SMS 단체 발송
  7. 발송 메시지: "[워크위드] {현장명} 현장 금일 인원이 필요합니다. 출근 가능하시면 회신 부탁드립니다."
팁: 충주 지역 현장은 지역 필터를 "충주"로 설정하면 당일 출근 가능한 인력만 표시됩니다. 먼 지역 사람에게 문자 보내도 응답률이 낮으니 지역 필터 활용하세요.

? 자주 묻는 질문

근로자가 폰을 바꾸면?

관리자가 구글시트 "근로자" 탭의 E열(기기ID) 셀을 비우면 다음 체크인 시 새 기기로 자동 등록됩니다.

동명이인이 있으면?

대시보드에 노란색 경고가 표시됩니다. 투입현황에 이름 뒤 번호를 붙여주세요 (예: 김철수1, 김철수2). 체크인은 토큰 기반이라 구분됩니다.

GPS가 안 잡히면?

체크인 페이지 하단의 "GPS가 안 되면 여기를 눌러주세요" → 수동 체크인 (위치 없이 시간만 기록, 상태: "수동")

비용은?

GitHub Pages + Google Apps Script + 구글시트 = 전부 무료. SMS 발송만 알리고 충전금 사용.
